Dynamic Cloud Simulation 
using Cellular Automata

CoffeeScript + WebGL
Based on 
'A Simple, Efficient Method for Realistic Animation of Clouds' 
by Dobashi, Kaneda, Yamashita, Okita & Nishita
Good-looking clouds are incredibly important in creating  photo-realistic outdoor scenes:
Using Cellular Automata allows us to create a highly dynamic
simulation of cumulus-like clouds, in almost real-time,
by simplifying the highly complex, physical description of a
cloud-system into a set of simple Boolean operations.
The CA is set up as a 3-dimensional grid, where each cell in 
the grid holds three Boolean values:
  • hum - vapor/humidity
  • act - phase-transition/activation
  • cld - clouds
Initially, at each cell hum & act are randomly set to 1 or 0, and
 cld is set to 0.
The simulation then runs through three key stages:
  • Extinction - the process of cloud extinction:
As a bubble of air is heated by the underlying terrain, is becomes less dense,
and rises into regions of lower pressure, which allows the bubble to expand.
Expansion cools the bubble, which increases the relative humidity inside the
bubble. As the humidity increases, water vapour in the bubble goes through
 a phase-transition and become water droplets.
  • Growth - the process of clouds forming:
As a water droplets transition back to vapour, the reverse process occurs.
  • Advection - the process of movement caused by wind:
The original model assumes that wind moves in one direction, but does account
 for the change in wind-speed with respect to height.
Growth:
There are transition rules for each of hum, act & cld:
cld(x, y, z, t + 1) = cld(x, y, z, t) || act(x, y, z, t)
hum(x, y, z, t + 1) = hum(x, y, z, t) && !act(x, y, z, t)
act(x, y, z, t + 1) = !act(x, y, z, t) && hum(x, y, z, t) &&
                                       ƒact(x, y, z, t), where

ƒact = act(x + 1, y, z, t) || act(x, y + 1, z, t) || act(x, y, z + 1, t) || 
               act(x - 1, y, z, t) || act(x, y - 1, z, t) || act(x, y, z - 1, t) || 
               act(x - 2, y, z, t) || act(x + 2, y, z, t) || act(x, y - 2, z, t) ||
               act(x, y + 2, z, t) || act(x, y, z - 2, t)
Extinction:
There are again transition rules for each of hum, act & cld:
hum(x, y, z, t + 1) = hum(x, y, z, t) || IS(rnd < phum(x, y, z, t))
cld(x, y, z, t + 1) = cld(x, y, z, t) && IS(rnd > pext(x, y, z, t))
act(x, y, z, t + 1) = act(x, y, z, t) || IS(rnd < pact(x, y, z, t))
phum, pext & pact are user-defined values representing
the probabilities of humidity, extinction and
activation occuring.

IS is a Boolean function that simply evaluates the 
expression.

rnd is a pseudo-random number between 0 and 1
Advection:
And again, there are transition rules for hum, act & cld:
hum(x, y, z, t + 1) = if x - v(z) > 0 ? hum(x - v(z), y, z, t) : 0
cld(x, y, z, t + 1) = if x - v(z) > 0 ? cld(x - v(z), y, z, t) : 0
act(x, y, z, t + 1) = if x - v(z) > 0 ? act(x - v(z), y, z, t) : 0

where v(z) is a piecewise linear function that gives
wind-speed as a function of height.
Implementation:
Original paper implemented in C++.
JavaScript/CoffeeScript issues:
Bitwise operators:
No concept of a 'bit' in JS - Original implementation used 3 bits per cell.

Bitwise operators can be used on Numbers but these are truncated to 32 bits.

In order to manipulate an entire state (e.g. hum, act, cld) at once, they are stored
as Strings, - '10010001010100100101' - (three character per cell).

So it uses lots more memory, but this means we can have some interesting
String-based bitwise operations to makes things much faster:
logic =
    and: (str1, str2, res = '') ->
      (res += "#{str1[i] & str2[i]}") for i in [0...str1.length]
      res
    or: (str1, str2, res = '') ->
      (res += "#{str1[i] | str2[i]}") for i in [0...str1.length]
      res
    not: (str, res = '') ->
      (res += if str[i] is '0' then '1' else '0') for i in [0...str.length]
      res
Wee bit hacky, but it's fast and it works!
Binary Grid to Continuous Distribution:
Boolean grid isn't useful when it comes to rendering, so we have to
generate a 3D continuous distribution function that describes the clouds:
density = toGrid.call(this, ((x, y, z) -> getDensity(@_.clouds, x, y, z)))

toGrid = (func) ->
    g = []
    for x in [0...@x]
      g[x] ?= []
      for y in [0...@y]
        g[x][y] ?= []
        for z in [0...@z]
          g[x][y][z] = func.call(this, x, y, z)
    g
getDensity = (clouds, x, y, z) ->
    weight = (dx, dy, dz) ->
      Math.sqrt((dx * dx) + (dy * dy) + (dz * dz))
    d = 0
    if clouds[x]?[y]?[z] is 1
      for xs in [x - xSmooth..x + xSmooth]
        for ys in [y - ySmooth..y + ySmooth]
          for zs in [z - zSmooth..z + zSmooth]
            exists = clouds[xs]?[ys]?[zs]?
            d += (if exists and clouds[xs][ys][zs] is 1 then weight(xs - x, ys - x, zs - x) else 0)
    if d isnt 0
      d /= maxWeight
      d /= ((2 * xSmooth + 1) * (2 * ySmooth + 1) * (2 * zSmooth + 1))
    else
      d = d
    d

xSmooth = ySmooth = zSmooth = 2
maxWeight = 0
for x in [-xSmooth..xSmooth]
  for y in [-ySmooth..ySmooth]
    for z in [-zSmooth..zSmooth]
      maxWeight += Math.sqrt((x * x) + (y * y) + (z * z))
